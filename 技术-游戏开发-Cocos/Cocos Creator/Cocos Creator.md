Cocos Creator
# Cocos Creator 安装
1. 进入官网 Cocos.com
2. 进入网页中的"产品"选项
3. 下载Cocos DashBoard 
4. 在Cocos DashBoard安装Cocos Creator
# Hello World 项目
1. 在打开的Cocos Creator中点击新建项目
2. 新建一个HelloWorld项目,选择创建并打开
3. 之后再次打开此项目时,可以在Dashboard中的项目选项中找到此项目
4. Cocos Creator的项目不使用项目工程文件来打开项目 
## Hello World 项目中的文件夹
1. asserts文件夹:保存第三方创建音乐图片脚本等文件
   1. Scene:保存场景文件
   2. Script:脚本文件
   3. Texture:
2. Hello World 背景文件被放置在assert/Scene文件夹下

# 从空项目到Hello World
## 创建文件树:
1. 资源管理器中assert文件夹下建立Scene文件夹;
2.  Scene文件夹下建立gameView场景文件(Scene);
## 场景文件的属性
创建好场景文件后在层级管理器中即生成了两个节点(形成一个层级):
- Canvas:**画布,表示窗口大小**,包含屏幕分辨率属性以及窗口是否与显示器的横/高适配的属性;
  - Main Camera:主摄像机,位与Canvas层级下的子节点;

# 节点
- 所有物体都是由节点所构成的,节点中属性描述控件物理属性(大小,位置等)
## 创建一个新节点
- 所有节点的创建都只需要在对应父节点上右键选择创建节点;

# 控件节点的分类:
## 基节点:
- 建立一个空节点,并将其更名为(baseView)基节点;
- 基节点的作用:盛放所有其他控件(节点)的节点;
- 基节点需要盛放所有控件节点,基节点的大小需要与画布大小相同;(Size属性)
## Button节点:
代表所有按钮控件;
### 创建Button节点:
1. Cocos Creator自动创建的Button节点:
   1. 层级管理器中右键创建节点->创建UI节点->Button
2. 手动创建的Button节点:
   1. 层级管理器中右键创建节点->创建空节点->添加组件(属性检查器中)->UI组件->Button
#### 两种方式创建结点的区别:
- 两种Button主体基本相同
- 手动创建的按钮节点的Button属性中一些小属性为空
- 自动创建的按钮节点的Button属性中的小属性均为默认值
#### Button结点的构成(父子层级关系)
- New Button    //新创建的按钮节点  
  - Background  //背景图片,指按钮上的图片
    - Label     //按钮标签,指按钮上的文字
- 注意:
1. 父节点的**父子节点共同属性**改变时(移动/大小变化/..),**子节点对应属性**会一同变化;
2. 子节点的**父子节点共同属性**改变时,**父节点对应属性**不一定变化
3. 子节点在层级管理器中的属性可以被锁定,使其与父结点相同;
#### New Button的属性组成:
1. Node(节点)相关属性:位置,旋转,大小,颜色....
2. Button相关属性:
   1. Target:
      1. 红×表示删除此属性的值
      2. 点击属性值会显示此属性在层级管理器中的位置(同名时区分多个按钮)
      3. 向此属性赋值时,将层级管理器中的Background直接拖拽至Target属性中即可完成赋值;
      4. 属性旁的小框显示了此属性应使用何种类型来赋值(Node),不可跨类型复制;
   2. resize to Target:(尽量不使用)
      1. 当子节点的**父子节点共同属性**与父节点不同时,父节点可以依据所选择的子节点大小重新确定尺寸
      2. 由于此时,子节点大小与父节点不同,,当父节点调整为子节点的大小时,子节点会跟据与父结点的大小差值进行变化,产生一直变化的现象;不易调整大小;
   3. Interactable:
      1. 使当前按钮可以使用,取消勾选则不可互动
   4. Transition: 按钮状态变化时的状态表现(枚举类型):
      1. SPRITE:
         1. Button 在 Normal 状态下的 SpriteFrame。
         2. Button 在 Normal 状态下的 SpriteFrame。
         3. Button 在 Normal 状态下的 SpriteFrame。
         4. Button 在 Normal 状态下的 SpriteFrame。
      2. NONE:
      3. COLOR:
      4. SCALE:
3. 属性旁的对勾关掉即可使这些属性失效(是按钮的interactable属性失效,从而使按钮无反应)
4. 属性的对应设置包含了如下操作
   1. 移除此分支属性(按钮中的Button属性)
   2. 重置属性
   3. 场景中上下移动按钮
   4. 复制组件(将此按钮的Button属性复制)
   5. 粘贴组件(将已复制好的属性添加至新的空节点中)
